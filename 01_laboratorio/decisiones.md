# Decisiones

> Este documento registra las decisiones estructurales del laboratorio.

No documenta opiniones.

Documenta acuerdos.

Cada decisión debe incluir el contexto que la originó.

---

# Formato

## D-XXXX

**Fecha**

**Estado**

* Activa
* Reemplazada
* En revisión

**Contexto**

...

**Decisión**

...

**Motivo**

...

**Consecuencias**

...

---

# D-0001

**Estado**

Activa

**Contexto**

Era necesario decidir si el proyecto comenzaría como una plataforma tecnológica o como una investigación.

**Decisión**

El laboratorio priorizará el desarrollo del método antes que el desarrollo del software.

---

# D-0002

**Estado**

Activa

**Contexto**

El alcance del proyecto crecía constantemente.

**Decisión**

El proyecto de la diplomatura será únicamente el MVP del laboratorio.

---

# D-0003

**Estado**

Activa

**Decisión**

La comunidad y el motor analítico serán tratados como productos diferentes.

---

# D-0004

**Estado**

Activa

**Decisión**

Toda idea que exceda el MVP será registrada en Masterplan antes de modificar el proyecto.

---

# D-0005

**Estado**

Activa

**Decisión**

El repositorio se organizará siguiendo el ciclo del conocimiento y no la estructura tradicional de un proyecto de software.
